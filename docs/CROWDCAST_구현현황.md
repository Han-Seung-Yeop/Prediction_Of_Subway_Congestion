# CROWDCAST 구현 현황 기획안
**타기 전에 미리 확인하는 칸별 혼잡도 예측 + 3D 지하철 지도 — 프론트엔드 프로토타입**

> 이 문서는 [`CROWDCAST_기획안.md`](./CROWDCAST_기획안.md)에서 정의한 서비스 컨셉을 바탕으로, 현재까지 실제로 구현된 프론트엔드 프로토타입의 범위와 구조를 정리한 문서입니다.

---

## 1. 한눈에 보기

| 항목 | 내용 |
|---|---|
| 형태 | Vite + React 18 + TypeScript SPA, 모바일 프레임 UI (데스크톱에서는 중앙 정렬된 카드) |
| 데이터 | 전부 목업(mock) — 노선 4개, 역 19개, 이벤트 3건 |
| 예측 로직 | 규칙 기반 추정 함수 (역 평균 × 칸별 가중치 + 이벤트 delta), 실제 ML 모델 미연동 |
| 3D 렌더링 | react-three-fiber + drei(Three.js) 로 3종의 3D 씬 구현 |
| 상태 관리 | 별도 라이브러리 없이 `App.tsx` 내 `useState`/`useMemo`로 관리 |
| 백엔드 | 없음 (전량 클라이언트 사이드) |

## 2. 화면 구성 (정보 구조)

```
TopBar (로고/시계)
 └ LineSelector (노선 선택: 2·9·4·7호선)
    └ StationSelector (역 선택, 노선별 대표 구간)
       └ Controls (상행/하행 방향 · 시간대 슬라이더 5~23시)
          └ ArrivalBanner (다음 열차 도착 예정 시간 — 목업)
             └ CongestionSummary (평균 혼잡도 · 이벤트 영향 · 추천 탑승칸 CTA)
                └ 시각화 영역 (ViewToggle: 3D 지도 / 칸 목록 / 역 지도)
                   ├─ Train3D      : 3D 열차 히트맵 (컷어웨이 뷰)
                   ├─ CarStrip     : 2D 칸별 막대 그래프
                   └─ Station3D    : 역 내부 3D 구조도 (역삼역 예시)
                └ CarDetail (선택 칸 상세 — 여유도 순위 · 구조물 · 내부 3D 버튼)
                └ EventForecast (이벤트 혼잡 예보 카드 목록 → 탭 시 해당 노선/역으로 이동)

CarInterior3D (모달) : 칸 내부 1인칭 3D 뷰 — 좌석/입석 인원, 여유 좌석 수, 칸 전환 칩
```

## 3. 기능별 구현 상세

### 3.1 노선·역·방향·시간대 선택 (`Selectors.tsx`, `Chrome.tsx`)
- 4개 노선(2·9·4·7호선) 가로 스크롤 선택, 노선 컬러 배지 적용
- 노선별 대표 역 구간 선택 (역 사이 진행 라인 시각화)
- 상행/하행(또는 내선/외선) 방향 토글 — 노선별 종착역 라벨 포함
- 5시~23시 시간대 슬라이더로 예측 시점 조정
- 노선 변경 시 해당 노선 첫 역·기본 방향으로 자동 이동

### 3.2 칸별 혼잡도 예측 (`data/predict.ts`)
- **시간대 기반 역 평균** `baseCongestionByHour(hour)`: 출근(8시) · 퇴근(18.5시) · 점심(13시) 피크를 가우시안 곡선으로 근사, 최대 82% 포화
- **칸별 가중치** `carWeights(stationId)`: 승강장 구조물(계단/환승/출구) 위치에 가우시안 커널을 적용해 인접 칸에 혼잡 쏠림 반영, 양 끝 칸은 소폭 할인, 평균 1.0으로 정규화
- **이벤트 델타 반영**: 해당 역에 걸린 이벤트 중 최댓값을 가중치 비례로 각 칸에 가산 (혼잡 칸일수록 더 크게 반영)
- **결정론적 노이즈**: 역ID+칸번호+시간 해시 기반 유사난수로 매 렌더 동일한 값 유지 (같은 입력 → 같은 결과)
- 5단계 혼잡도 레벨 산출: 여유(<30) · 보통(<50) · 주의(<70) · 혼잡(<85) · 매우혼잡(≥85), 각 레벨별 색상·설명 텍스트 정의
- 결과값: 칸별 배열, 평균, 이벤트 증가분, 최적/최혼잡 칸 번호

### 3.3 혼잡도 요약 & 추천 탑승칸 (`Summary.tsx`)
- 평균 혼잡도 큰 숫자 표시 + 레벨 배지
- 이벤트 영향이 있는 경우 "+N%p 이벤트 영향" 별도 카드
- 추천 탑승칸 CTA 버튼 → 탭하면 해당 칸의 내부 3D 뷰로 바로 진입
- 하단 범례(5단계 색상 · 라벨)

### 3.4 3D 열차 히트맵 (`Train3D.tsx`)
- 10량 편성 열차를 컷어웨이(단면 개방) 형태로 3D 렌더링, 실제 진행 방향대로 배치
- 각 칸의 혼잡도 값을 승객 수(InstancedMesh)로 환산해 좌석 우선 → 입석 순으로 배치
- 칸 상단에 호칸 번호 + 혼잡도(%) + 레벨 라벨을 말풍선(HTML overlay)으로 표시
- 최여유 칸에 초록 마커 표식
- 선택된 칸은 살짝 부양(y축 애니메이션) + 발광 효과
- 드래그 회전 / 핀치 확대 / 탭으로 칸 선택 (드래그와 탭 제스처를 거리·시간 임계값으로 구분)
- 레일 + 침목 + 승강장 + 안전선까지 배경 구조물로 구현

### 3.5 2D 칸 목록 뷰 (`CarStrip.tsx`)
- 3D 렌더링 없이 가벼운 막대 그래프로 칸별 혼잡도 표시 (진행방향 라벨 포함)
- 혼잡도 값에 비례한 막대 높이, 구조물 아이콘(계단/환승/출구) 오버레이
- 최여유 칸에 pulse 마커

### 3.6 칸 상세 정보 (`CarDetail.tsx`)
- 선택 칸의 혼잡도 수치·레벨·설명, 전체 칸 중 여유도 순위
- 혼잡도 게이지 바
- 해당 칸에 인접한 승강장 구조물 태그 (계단/환승/출구/엘리베이터) + "구조물 인접 · 혼잡 쏠림 주의" 안내
- "칸 내부 3D로 보기" 버튼 → `CarInterior3D` 모달 오픈

### 3.7 칸 내부 3D 뷰 (`CarInterior3D.tsx`) — *신규 구현*
- 실제 전동차 내부(좌석/손잡이/천장 덕트/문/안내 디스플레이)를 1인칭 시점으로 재현
- 혼잡도 값에 따라 좌석 착석 인원 + 입석 인원 수를 계산해 InstancedMesh로 배치
- 여유 좌석 수, 입석 밀집도(여유/보통/다소높음/높음/매우높음) 통계 패널
- 하단 칩으로 다른 칸으로 즉시 전환 가능 (전체 칸 혼잡도 미리보기 포함)
- OrbitControls로 제한된 각도 내에서 둘러보기/확대 가능

### 3.8 역 내부 3D 지도 (`Station3D.tsx`) — *신규 구현*
- 2호선 역삼역 안내도를 참고해 하드코딩한 3층 구조(B1·B2 대합실, B3 승강장) 3D 모델
- 출구 8개 계단/에스컬레이터 위치, 엘리베이터, 개찰구, 편의시설(화장실·소화전·AED·비상전화·구호물품·발매기), 고객안내센터·만남의장소·현위치 마커, 최단 대피경로 표시
- 층별 반투명 데크 + 구조 기둥 + 승강장 트랙/안전선/점자블록까지 디테일 구현
- 자동 회전 카메라 + 드래그/줌 조작, 범례 오버레이
- 현재는 역삼역 1개 역만 예시로 하드코딩 (다른 역에는 미적용)

### 3.9 이벤트 혼잡 예보 (`EventForecast.tsx`)
- 공연(콘서트)·스포츠·페스티벌 3개 목업 이벤트 카드 표시
- 혼잡 증가분(%p)에 따른 심각도 색상 구분 (급증/증가/주의)
- 카드 탭 시 해당 이벤트가 걸린 노선/역으로 화면 자동 전환

### 3.10 성능 최적화
- 3D 컴포넌트(Train3D, CarInterior3D, Station3D) 는 `React.lazy` + `Suspense`로 지연 로딩해 초기 번들 크기 최소화 (three.js 번들 분리)
- 각 로딩 상태에 스피너 + 안내 텍스트 표시

## 4. 데이터 모델 (`data/types.ts`, `data/subway.ts`)

| 타입 | 설명 |
|---|---|
| `Line` | 노선 ID·표기명·라벨·색상·순환 여부 |
| `Station` | 역 ID·이름·소속 노선·순서·환승 노선 목록 |
| `PlatformFeature` | 승강장 구조물(계단/출구/환승/엘리베이터) 위치 — 칸 쏠림 원인 데이터 |
| `CarCongestion` | 칸 번호·예측 혼잡도(0~100)·레벨 |
| `Direction` | 상행/하행 방향·종착역 라벨 |
| `CrowdEvent` | 이벤트 타입·제목·장소·연계 역/노선·시각·혼잡 증가분(%p) |

**목업 데이터 범위**
- 노선 4개(2·9·4·7호선), 역 19개 (노선별 대표 구간만)
- 승강장 구조물 상세 데이터는 강남·잠실·고속터미널 3개 역만 지정, 나머지는 `DEFAULT_FEATURES`로 대체
- 이벤트 3건: 아이유 콘서트(강남), LG-두산 야구(잠실), 한강 불꽃축제(여의도)

## 5. 기술 스택 & 아키텍처

- **빌드/프레임워크**: Vite 5, React 18, TypeScript 5
- **스타일**: Tailwind CSS (다크 테마 고정, `ink-*`/`crowd-*` 커스텀 컬러 팔레트)
- **3D**: `@react-three/fiber` + `@react-three/drei` (OrbitControls, RoundedBox, Html, ContactShadows) + `three`
- **공통 3D 유틸** (`lib/three.ts`): 사람 모형 지오메트리, 시드 기반 의사난수(mulberry32), 등간격 배열(linspace), 색상 shade 헬퍼
- **상태 관리**: 컴포넌트 로컬 상태만 사용 (Redux/Zustand 등 미도입)
- **라우팅**: 없음 (단일 화면, 내부 상태로 뷰 전환)

```
src/
├─ data/
│  ├─ types.ts      # 도메인 타입
│  ├─ subway.ts     # 노선/역/승강장/이벤트 목업 + 조회 헬퍼
│  └─ predict.ts     # 칸별 혼잡도 추정 로직
├─ lib/three.ts      # 3D 공통 유틸
├─ components/
│  ├─ Chrome.tsx        # 상단바 · 도착 배너 · 뷰 토글(3D/2D/역지도)
│  ├─ Selectors.tsx     # 노선 · 역 · 방향 · 시간대 선택
│  ├─ Summary.tsx       # 평균 혼잡도 요약 · 추천 칸 · 범례
│  ├─ CarStrip.tsx      # 2D 칸별 막대 뷰
│  ├─ CarDetail.tsx     # 선택 칸 상세
│  ├─ Train3D.tsx       # 3D 열차 히트맵
│  ├─ CarInterior3D.tsx # 칸 내부 1인칭 3D 뷰 (모달)
│  ├─ Station3D.tsx     # 역 내부 3D 구조도
│  ├─ EventForecast.tsx # 이벤트 혼잡 예보 카드
│  └─ icons.tsx         # SVG 아이콘
└─ App.tsx           # 화면 조립 · 상태 관리
```

## 6. 원 기획안 대비 진행 상황

| 기획안 항목 | 상태 | 비고 |
|---|---|---|
| 칸별 혼잡도 사전 예측 | ✅ 구현 | 규칙 기반 추정 (ML 모델은 미연동) |
| 3D 색상 히트맵 시각화 | ✅ 구현 | Train3D, 계획보다 확장(칸 내부 3D까지 구현) |
| 이벤트 변수 반영 예측 | ✅ 부분 구현 | 목업 delta 값만 반영, 실제 학습 모델(XGBoost/LightGBM) 미연동 |
| 3D 렌더링 기술 스택 선정 | ✅ 완료 | react-three-fiber + drei로 확정 |
| UI/UX 와이어프레임 | ✅ 실제 구현으로 대체 | 프로토타입이 곧 UI 시안 역할 |
| 역 내부 구조 3D 지도 | ✅ 구현 (범위 초과 달성) | 원 기획안에 없던 기능, 역삼역 1개 예시로 구현 |
| 공공 API 연동 (실시간 도착정보 등) | ❌ 미구현 | 전량 목업 데이터 |
| KOPIS·스포츠 데이터 기반 실제 예측 모델 | ❌ 미구현 | 이벤트 데이터도 3건 하드코딩 |
| 칸별 가중치 실측 데이터 확보 | ❌ 미구현 | 구조물 위치 기반 추정 로직만 존재 |
| 전체 노선/역 커버리지 | ❌ 미구현 | 4개 노선 · 19개 역만 지원 |

## 7. 현재 한계

- **데이터**: 모든 데이터(노선·역·구조물·이벤트·혼잡도)가 하드코딩된 목업이며 실제 공공데이터포털 API·실시간 도착정보와 연동되어 있지 않음
- **예측 정확도**: 혼잡도 예측은 실제 통계 학습이 아닌 수식 기반 추정치로, 실사용 정확도는 검증되지 않음
- **역 커버리지**: 역 내부 3D 지도는 역삼역 1개만 구현되어 있어 다른 역 선택 시에도 동일한 구조도가 표시됨
- **백엔드 부재**: 서버/DB/인증 등 백엔드 계층이 전혀 없는 프론트엔드 단독 프로토타입
