# CROWDCAST 기획안
**타기 전에 미리 확인하는 칸별 혼잡도 예측 + 3D 지하철 지도**

---

## 1. 문제 정의: 탑승 전 혼잡도 정보의 부재

실시간·칸 단위 혼잡도 정보를 알 수 없는 상태의 지하철 탑승은 어느 칸이 붐빌지에 대한 불확실성을 키워 이동 스트레스로 이어진다. 혼잡도를 사전에 인지할 수 있는 경험은 이동에 대한 통제감을 높인다.

---

## 2. Interview & Research

**조사 대상**: 출퇴근 시 지하철을 이용하는 20~30대 직장인·통학생 5명
**목적**: 칸별 혼잡도 정보 부재로 인한 불편 경험과 니즈 파악

### Interview Findings
- "타기 전엔 어느 칸이 덜 붐빌지 전혀 알 수가 없어요. 그냥 눈앞에 오는 칸에 밀려 타죠."
- "붐비는 날은 어느 칸이든 다 꽉 차 보여서, 그나마 나은 칸이 어딘지 감이 안 와요."
- "지하철 앱은 도착 시간만 알려주지, 어느 칸이 붐비는지는 안 알려주잖아요."
- "콘서트·야구 있는 날은 평소랑 완전 달라요. 미리 알았으면 시간대를 바꿨을 텐데."
- "퍼센트로 몇 %라고 하면 잘 안 와닿아요. 그림으로 보이면 훨씬 낫겠죠."

> **핵심 발견**: 탑승 전 칸별 혼잡도를 미리·직관적으로 알고 싶다는 니즈 확인
> → **탑승 전 칸별 혼잡도 정보의 부재로 인한 이동 불확실성**

---

## 3. Needs & Feature

인터뷰 결과를 종합하여 Pain point를 도출하고, 니즈 키워드와 해결 기능으로 연결.

| Pain Point | 니즈 키워드 | Solution |
|---|---|---|
| 타기 전 어느 칸이 덜 붐빌지 알 수 없음 | 정확성 | 칸별 혼잡도 사전 예측 제공 |
| 그나마 나은 칸이 어딘지 감이 안 옴 | 비교성 | 칸 간 혼잡도 차이 한눈에 표시 |
| 기존 앱은 도착 시간만 있고 칸 정보 없음 | 접근성 | 도착 정보를 넘어선 칸 단위 안내 |
| 이벤트·요일에 따라 혼잡도가 급변 | 예측성 | 이벤트 변수 반영 혼잡도 예측 |
| 퍼센트 숫자로는 혼잡 정도가 안 와닿음 | 직관성 | 3D 색상 히트맵 시각화 |

## 4. 기존 서비스 비교

이미 칸별 혼잡도를 제공하는 앱(또타지하철·TMAP·지하철종결자 등)이 있으나, **3D 시각화**와 **이벤트 예측**에서 차별점을 가진다.

| 비교 항목 | 기존 앱 | CROWDCAST |
|---|---|---|
| 칸별 혼잡도 표시 | ○ 평면 색상 바 | ○ 제공 |
| 3D 내부 지도 | ✕ 2D 표시만 | ● 3D 히트맵 |
| 이벤트 기반 예측 | ✕ 통계·현재값만 | ● 공연·경기 반영 |
| 직관적 공간 파악 | △ 숫자·색상 중심 | ● 실제 열차 형태 |

> 기존 앱의 2D·예측 부재 한계를 넘어선 차별점 확보

## 5. Scenario (대표 시나리오 3가지)

### ① Pre-boarding Check
`#탑승전확인` `#칸별예측`
승강장에서 곧 들어올 열차의 칸별 예측 혼잡도를 확인하고, 가장 여유로운 칸 앞에서 대기해 붐비지 않게 탑승한다.

### ② 3D Congestion View
`#3D시각화` `#직관적파악`
3D 지하철 지도로 열차 내부를 실제 형태로 살펴본다. 붐비는 칸은 빨강, 여유로운 칸은 초록으로 표시돼 빠르게 파악한다.

### ③ Event Forecast
`#이벤트예측` `#사전대비`
이용 노선 인근 경기·공연 종료 시간대를 반영해 '평소 대비 혼잡 예상'을 미리 제공, 출발 시간을 조정해 혼잡을 피한다.
(예시: 아이유 콘서트 - 강남역·오늘 19:00, 혼잡 급증 예상 +42%p / LG 트윈스 vs 두산 - 잠실역·내일 18:30, 혼잡 증가 예상 +28%p)

## 6. Technical Research

### 데이터의 한계
- 공공데이터포털의 혼잡도 통계는 **역 단위 평균만** 제공, 칸 단위 분포 데이터는 비공개
- 상용 서비스(SK PUZZLE)조차 칸별 평균까지만 제공

### 칸별 혼잡도 추정 로직
```
역 단위 평균 혼잡도 × 칸별 가중치 패턴 = 칸 단위 혼잡도 추정
```
- 계단·환승구 위치에 따른 칸 쏠림 현상을 가중치로 반영
- 특정 칸이 상습적으로 붐비는 실제 경향을 패턴화

### 이벤트 변수 예측 로직
- **데이터셋 구축**: KOPIS(공연) + 스포츠 관중 데이터 크롤링 → 이벤트 이력 데이터셋
- **학습 신호**: 이벤트 당일 혼잡도 증가분 (delta = 이벤트일 − 동일 요일 평균)
- **피처**: 공연장 수용인원, 이벤트 유형, 요일, 날씨
- **모델**: XGBoost / LightGBM 회귀 모델 → 예정 이벤트의 혼잡 증가분 예측

> 역단위 데이터 한계를 보완하는 칸별 혼잡도 추정 방법론의 확립

## 7. Service Concept & Key Feature

**CROWDCAST** — 타기 전에 미리 확인하는 칸별 혼잡도 예측 + 3D 지하철 지도

### Key Feature ① 칸별 혼잡도 예측
역 단위 데이터를 칸별 가중치로 세분화하고 이벤트 변수까지 반영해 각 칸의 혼잡도를 예측한다. 노선·역·시간대를 선택해 어느 칸이 여유로운지 타기 전에 미리 확인할 수 있다.

### Key Feature ② 지하철 내부 3D 지도
열차 내부를 3D 모델로 구현하고 예측된 칸별 혼잡도를 색상 히트맵(초록·노랑·빨강)으로 매핑한다. 평면 수치가 아닌 실제 열차 형태 위에서 직관적으로 혼잡 분포를 파악할 수 있다.

> 예측과 3D 시각화를 결합한 직관적 혼잡도 안내
> *(key feature 내용은 추후 추가 예정)*

## 8. Service Flow

### User Flow (사용자 이용 흐름)
```
앱 실행 → 노선·역 선택 → 칸별 예측 혼잡도 확인 → 3D 지도로 상세 파악 → 여유 칸 탑승
```

### Data Flow (데이터 처리 흐름)
```
공공 API 수집          →  칸별 추정 로직        →  이벤트 예측 모델         →  3D 렌더링            →  사용자 화면
(혼잡도 통계 +             (역 평균 ×               (KOPIS·스포츠               (칸별 혼잡도             (예측 결과
 실시간 도착정보)           칸별 가중치)             delta 회귀 예측)            색상 매핑)               3D 지도 제공)
```

> 사용자 흐름과 데이터 흐름을 아우르는 서비스 구조 정립
